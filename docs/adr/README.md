# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for EZ Agents.

## What is an ADR?

An Architecture Decision Record (ADR) is a document that captures an important architectural decision made along with its context and consequences.

### ADR Structure

Each ADR contains:
- **Title** - Short descriptive name
- **Status** - Proposed, Accepted, Deprecated, or Superseded
- **Context** - Problem statement and background
- **Decision** - The change being proposed
- **Consequences** - Impact of the decision
- **Related Records** - Links to other ADRs

### Status Values

- **Proposed** - Under discussion, not yet decided
- **Accepted** - Decision made, being implemented
- **Deprecated** - No longer recommended, use alternative
- **Superseded** - Replaced by a newer ADR
