<action>

2. Replace interface usage with concrete type:
   ```typescript

     validate(email: string): boolean;
   }

   class EmailValidator implements IEmailValidator {
     validate(email: string): boolean { ... }
   }

   function processEmail(validator: IEmailValidator) { ... }

     validate(email: string): boolean { ... }
   ```

   - Remove `implements InterfaceName` from classes
   - Update function parameter types from interface to concrete class

5. Update barrel exports (`bin/lib/index.ts`):
   - Remove interface export
   - Keep concrete class export

6. For interfaces marked "Keep with deprecation":
   - Add JSDoc deprecation notice:
     ```typescript
     /**
      * @deprecated Will be removed in v7.0.0. Use ConcreteClass instead.
      */
     export interface OldInterface { ... }
     ```

7. Update tests to use concrete types instead of interfaces
</action>

### Task 11.3-D: Remove Unused Parameters

- The actual source files containing unused parameters
</read_first>

<action>
1. For each function with unused parameters marked "Remove":

2. Remove unused parameters from function signature:
   ```typescript

   function processData(data: string, options: ProcessOptions, logger: Logger): string {
     // options and logger never used

   }

   // AFTER

     return data.toUpperCase();
   }
   ```

3. Update all call sites:
   - Remove arguments from function calls
   - Update any code that constructs argument objects

4. For unused options object properties:
   - Remove unused properties from options interface
   - Update all code that constructs options objects

5. For functions marked "Deprecate":
   - Add deprecation notice to function JSDoc
   - Create new function without unused parameters
   - Mark old function as deprecated, pointing to new function
   ```typescript
   /**
    * @deprecated Use processDataV2 instead. The options parameter was never used.
    */
   function processData(data: string, _options?: ProcessOptions): string {
     return data.toUpperCase();
   }

   function processDataV2(data: string): string {
     return data.toUpperCase();
   }
   ```

6. Update tests to match new function signatures
</action>

### Task 11.3-F: Remove Over-Engineered Patterns

<read_first>
- .planning/phases/11-core-library-clean-code/reports/11.3-ABSTRACTIONS.md (removal list)
- The actual source files containing over-engineered patterns
</read_first>

<action>
1. For each abstraction marked "Remove":

2. Replace abstract class with concrete implementation:
   ```typescript
   // BEFORE
   export abstract class BaseProcessor {
     abstract process(data: string): string;
     protected validate(data: string): boolean { ... }
   }

   export class StringProcessor extends BaseProcessor {
     process(data: string): string { ... }
   }

   // AFTER
   export class StringProcessor {
     process(data: string): string { ... }
     private validate(data: string): boolean { ... }
   }
   ```

3. Move protected methods from abstract base to concrete class as private

4. Update all references to use concrete class directly

5. For hierarchies marked "Simplify":
   - Reduce inheritance depth from 3+ levels to max 2 levels
   - Merge base classes where appropriate
   - Use composition instead of inheritance where it simplifies

6. Update barrel exports and tests

7. Document all removals in commit messages with YAGNI rationale
</action>

- All type hierarchies marked "Simplify" reduced to max 2 levels
