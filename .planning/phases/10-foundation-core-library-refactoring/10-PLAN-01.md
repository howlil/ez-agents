- tsconfig.json (current TypeScript configuration)
- bin/lib/logger.ts (existing Logger class for decorator usage)
</read_first>

<action>
Create decorator infrastructure in new `bin/lib/decorators/` directory with these files:

1. **bin/lib/decorators/types.ts** — Type definitions:
   - `LogExecutionOptions` interface with properties: `logParams?: boolean`, `logResult?: boolean`, `logDuration?: boolean`, `level?: 'info' | 'debug' | 'trace' | 'warn' | 'error'`, `paramName?: string`
   - `CacheResultOptions` interface with properties: `ttl?: number`, `cacheKeyFn?: (...args: any[]) => string`
   - `ValidateInputOptions` interface with properties: `validatorFn: (...args: any[]) => void`

2. **bin/lib/decorators/LogExecution.ts** — Method execution logging decorator:
   - Export function `LogExecution(methodName: string, options?: LogExecutionOptions)`
   - Returns decorator that wraps method to log entry/exit/duration
   - Handle both sync and async methods (Promise detection)
   - Use `logger` from `../logger.js` with configurable log level
   - Log format: `[ClassName] Entering/Completed/Failed methodName` with duration and optional params/result

3. **bin/lib/decorators/CacheResult.ts** — Method result caching decorator:
   - Export function `CacheResult(cacheKeyFn: (...args: any[]) => string, ttl?: number)`
   - Default TTL: 300000ms (5 minutes)
   - Uses internal `Map<string, { value: any; expiry: number }>` for cache storage
   - Return cached value if not expired, otherwise execute and cache
   - Log cache hit/miss at debug level

4. **bin/lib/decorators/ValidateInput.ts** — Input validation decorator:
   - Export function `ValidateInput(validatorFn: (...args: any[]) => void)`
   - Execute validator before method, throw if validator throws
   - Pass through method result on success

5. **bin/lib/decorators/index.ts** — Barrel export:
   - Export all three decorators
   - Export all type definitions
</action>
