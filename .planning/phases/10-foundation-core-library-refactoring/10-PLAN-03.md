# Plan 10.3: Apply Observer Pattern for Event-Driven Modules (CORE-04)

**Requirement:** CORE-04 — Apply Observer pattern for event-driven modules

**Goal:** Implement EventEmitter-based Observer infrastructure to enable decoupled event handling for session state changes, phase transitions, and skill events.

### Task 1: Define Observer Pattern Types

<read_first>
- .planning/phases/10-foundation-core-library-refactoring/10-RESEARCH.md (Observer Pattern section)
- bin/lib/observer/index.ts (existing observer infrastructure)
</read_first>

<action>
Create type definitions at `bin/lib/observer/types.ts`:

1. **Event<T>** interface:
   - `name: string` — Event name (e.g., 'session:start')
   - `payload: T` — Event payload data
   - `timestamp: number` — Unix timestamp of event

2. **EventHandler<T>** type:
   - Function signature: `(payload: T, event: Event<T>) => void | Promise<void>`

3. **Observer<T>** interface:
   - `update(event: Event<T>): void | Promise<void>` — Method called when event occurs

4. **EventMap** interface for type-safe event registry:
   - Define all event names and their payload types
   - Example: `'session:start': SessionStartEvent`

5. Event payload interfaces:
   - **SessionStartEvent**: `{ sessionId: string; timestamp: number; phase?: string }`
   - **SessionStopEvent**: `{ sessionId: string; timestamp: number; duration: number }`
   - **PhaseStartEvent**: `{ phaseNumber: string; phaseName: string; timestamp: number }`
   - **PhaseCompleteEvent**: `{ phaseNumber: string; phaseName: string; duration: number; plansCompleted: number }`
   - **SkillTriggerEvent**: `{ skillName: string; trigger: string; context: string }`
</action>
